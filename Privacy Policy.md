## Privacy Policy – Release Scheduler for Jira

**Last Updated:** March 2026

### Overview
Release Scheduler for Jira ("the App") is designed to help teams manage release approvals and scheduling workflows within Jira and Slack. We take user privacy seriously and only collect the minimum data necessary to provide functionality.

---

### Data We Collect

The App may process the following data:

- Jira issue data (issue keys, release identifiers, workflow states)
- User identifiers (Jira account ID, Slack user ID)
- Release approval actions (approve/decline decisions)
- Slack workspace identifiers (team ID)
- Basic metadata required for integration (timestamps, request logs)

---

### Data We Do NOT Collect

We do not collect or store:

- User passwords
- Personal emails beyond what is provided by Jira/Slack APIs
- Payment or billing information
- Sensitive personal data

---

### How Data Is Used

We use data strictly to:

- Process release approval workflows
- Send Slack notifications and receive user responses
- Update Jira issues or release states
- Maintain audit logs for actions taken

---

### Data Storage

- Data is processed using Atlassian Forge infrastructure where applicable
- Temporary data may be stored for workflow processing
- No long-term storage of sensitive user data is performed unless explicitly required

---

### Third-Party Services

The App integrates with:

- Atlassian Jira Cloud
- Slack API

Data shared with these services is limited to what is required for functionality.

---

### Data Retention

- Action logs may be retained for audit purposes
- No unnecessary long-term storage of user data
- Data can be deleted upon request

---

### Security

We implement:

- Secure HTTPS communication
- Token-based authentication (Slack + Jira)
- Validation of incoming requests (Slack signing secret, shared secrets)
- Principle of least privilege for API scopes

---

### User Control

Users can:

- Revoke access via Jira or Slack
- Uninstall the app at any time
- Request data deletion
