## Compliance & Data Handling – Release Scheduler for Jira

### Overview

Release Scheduler for Jira is designed to align with common enterprise security and compliance expectations.

---

### GDPR

- No unnecessary personal data is stored
- Users may request data deletion
- Data is processed only for application functionality

---

### Data Residency

- Data is processed within Atlassian Forge infrastructure where applicable
- Slack data is processed via Slack’s infrastructure

---

### Data Minimization

We only process:

- Jira issue identifiers
- Slack user IDs
- Approval decisions

No excessive or unrelated data is collected.

---

### Access Control

- Access is limited to authorized users within Jira and Slack
- No external sharing of customer data

---

### Third-Party Compliance

We rely on:

- Atlassian Cloud compliance standards
- Slack security and compliance framework

---

### Data Ownership

All customer data remains the property of the customer.

---

### Uninstallation

When the app is uninstalled:

- Access tokens are revoked
- No further data processing occurs
